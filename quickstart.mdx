# Quickstart

This guide will walk you through creating your first Hyperterse MCP server, from initialization to making your first MCP request.

## Initialize a project

<Steps>
  <Step title="Create a new project">
    Run the init command to scaffold a starter project:

    ```bash
    hyperterse init
    ```

    This creates the following structure:

    ```text
    my-project/
      .hyperterse
      app/
        adapters/
          my-database.terse
        tools/
          hello-world/
            config.terse
            user-data-mapper.ts
    ```
  </Step>

  <Step title="Understand the project structure">
    Hyperterse uses a convention-based structure:

    - **`.hyperterse`** - Root configuration file with server settings
    - **`app/adapters/`** - Database adapter configurations
    - **`app/tools/`** - Tool definitions (each tool in its own folder)

    <Tip>
    Each tool folder must contain a `config.terse` file. Optional TypeScript files can be added for custom logic.
    </Tip>
  </Step>

  <Step title="Examine the root config">
    The `.hyperterse` file defines your server configuration:

    ```yaml .hyperterse
    name: demo-framework
    version: 2.0.0-beta-1

    server:
      port: 8080
      log_level: 3
    ```

    This sets up an MCP server running on port 8080.
  </Step>

  <Step title="Check the adapter config">
    Adapters connect to your data sources. Here's the Postgres adapter:

    ```yaml app/adapters/my-adapter.terse
    connector: postgres
    connection_string: postgresql://demo:demo@localhost:5432/demo
    options:
      sslmode: disable
    ```

    <Note>
    Update the connection string to match your database credentials. You can also skip database setup and use script-backed tools instead.
    </Note>
  </Step>

  <Step title="Review the scaffolded tool">
    The init command creates a sample tool at `app/tools/hello-world/config.terse`:

    ```yaml app/tools/hello-world/config.terse
    description: "Get user by id, with input validation and response mapping."
    use: my-adapter
    statement: |
      SELECT * FROM users WHERE id = {{ inputs.userId }}
    inputs:
      userId:
        type: int
        description: "User ID to fetch."
    mappers:
      input: "./user-input-validator.ts"
      output: "./user-data-mapper.ts"
    auth:
      plugin: allow_all
    ```

    This tool queries a database using the `my-adapter` connection, validates inputs, and transforms outputs.
  </Step>
</Steps>

## Start the server

<Steps>
  <Step title="Run the development server">
    Start Hyperterse with:

    ```bash
    hyperterse start
    ```

    For automatic reloading during development:

    ```bash
    hyperterse start --watch
    ```

    You should see output indicating the server is running on `http://localhost:8080`.
  </Step>

  <Step title="Test the heartbeat endpoint">
    Verify the server is running:

    ```bash
    curl http://localhost:8080/heartbeat
    ```

    You should receive a 200 OK response.
  </Step>
</Steps>

## Make MCP requests

<Steps>
  <Step title="List available tools">
    Use the MCP `tools/list` method to see all registered tools:

    ```bash
    curl -X POST http://localhost:8080/mcp \
      -H "Content-Type: application/json" \
      -d '{
        "jsonrpc": "2.0",
        "method": "tools/list",
        "id": 1
      }'
    ```

    Response:

    ```json
    {
      "jsonrpc": "2.0",
      "id": 1,
      "result": {
        "tools": [
          {
            "name": "hello-world",
            "description": "Get user by id, with input validation and response mapping.",
            "inputSchema": {
              "type": "object",
              "properties": {
                "userId": {
                  "type": "integer",
                  "description": "User ID to fetch."
                }
              },
              "required": ["userId"]
            }
          }
        ]
      }
    }
    ```
  </Step>

  <Step title="Call a tool">
    Invoke the `hello-world` tool using the `tools/call` method:

    ```bash
    curl -X POST http://localhost:8080/mcp \
      -H "Content-Type: application/json" \
      -d '{
        "jsonrpc": "2.0",
        "method": "tools/call",
        "params": {
          "name": "hello-world",
          "arguments": {
            "userId": 1
          }
        },
        "id": 2
      }'
    ```

    Response:

    ```json
    {
      "jsonrpc": "2.0",
      "id": 2,
      "result": {
        "content": [
          {
            "type": "text",
            "text": "[{\"trace_id\":\"a1b2c3d4...\",\"id\":1,\"name\":\"John Doe\",\"email\":\"john@example.com\",\"created_at_iso\":\"2024-01-15T10:30:00.000Z\"}]"
          }
        ]
      }
    }
    ```

    <Note>
    This assumes you have a `users` table with the appropriate schema. If you don't have a database set up, create a script-backed tool instead.
    </Note>
  </Step>
</Steps>

## Create a custom tool

Let's create a script-backed tool that doesn't require a database:

<Steps>
  <Step title="Create a new tool directory">
    ```bash
    mkdir -p app/tools/get-weather
    ```
  </Step>

  <Step title="Create the tool config">
    Create `app/tools/get-weather/config.terse`:

    ```yaml app/tools/get-weather/config.terse
    description: "Get current weather for a city."
    handler: "./weather-handler.ts"
    auth:
      plugin: allow_all
    ```
  </Step>

  <Step title="Create the handler script">
    Create `app/tools/get-weather/weather-handler.ts`:

    ```typescript app/tools/get-weather/weather-handler.ts
    import dayjs from "dayjs";

    type Inputs = {
      city?: string;
      units?: "metric" | "imperial";
    };

    export default async function handler(payload: { inputs?: Inputs }) {
      const city = payload?.inputs?.city ?? "Bengaluru";
      const units = payload?.inputs?.units ?? "metric";

      const sampleTemp = units === "imperial" ? 77 : 25;
      const sampleWind = units === "imperial" ? "8 mph" : "13 km/h";

      return [
        {
          city,
          units,
          observed_at: dayjs().toISOString(),
          weather: "Partly cloudy",
          temperature: sampleTemp,
          wind: sampleWind,
          source: "demo-handler"
        }
      ];
    }
    ```
  </Step>

  <Step title="Test your new tool">
    The server will automatically pick up the new tool (if running with `--watch`). List tools again:

    ```bash
    curl -X POST http://localhost:8080/mcp \
      -H "Content-Type: application/json" \
      -d '{
        "jsonrpc": "2.0",
        "method": "tools/list",
        "id": 1
      }'
    ```

    You should see both `hello-world` and `get-weather` in the response.
  </Step>
</Steps>

## Build for production

When you're ready to deploy:

<Steps>
  <Step title="Validate your configuration">
    ```bash
    hyperterse validate
    ```

    This checks all config files and TypeScript scripts for errors.
  </Step>

  <Step title="Build a deployable artifact">
    ```bash
    hyperterse build -o dist
    ```

    This creates a `dist/` directory with everything needed to run in production.
  </Step>

  <Step title="Serve the artifact">
    ```bash
    hyperterse serve dist/
    ```

    This runs the compiled artifact without needing the source files.
  </Step>
</Steps>
